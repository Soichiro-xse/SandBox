# NDLOCR-Lite 評価レポート

## 概要

国立国会図書館（NDL）が公開した軽量OCRツール「NDLOCR-Lite」を調査・実行・評価した結果をまとめる。

- 公開日: 2026年2月24日
- ソース: https://github.com/ndl-lab/ndlocr-lite
- ライセンス: CC BY 4.0

---

## 環境

| 項目 | 値 |
|---|---|
| OS | Linux (Ubuntu) |
| Python | 3.11.14 |
| デバイス | CPU のみ（GPU なし） |
| モデル | DEIMv2（レイアウト認識）+ PARSeq（文字認識）|

---

## インストール

```bash
git clone https://github.com/ndl-lab/ndlocr-lite
cd ndlocr-lite
pip install -r requirements.txt
```

主な依存パッケージ:
- `onnxruntime==1.23.2` （GPU不要の推論エンジン）
- `opencv-python-headless==4.11.0.86`
- `pillow==12.1.1`
- `numpy==2.2.2`

インストールは問題なく完了（pyparsing のシステムパッケージ衝突を `--ignore-installed` で回避）。

---

## テスト実行

リポジトリ付属の3枚のサンプル画像で検証。すべて国立国会図書館デジタルコレクション由来の画像。

```bash
python3 src/ocr.py --sourceimg <image> --output <outdir> --viz True --device cpu
```

### 出力形式
- `.txt` テキスト（読み順整序済み）
- `.xml` 構造化 XML（座標・信頼度付き）
- `.json` JSON 形式
- `viz_*.jpg` 認識領域の可視化画像

文字単位の一致率（SequenceMatcher による）で評価。

## 誤認識の傾向分析

### digidepo_2531162（94.1%）
- OCR: `(2)気送子送付管` → 参照: `(ヱ)気送子送付管`（旧仮名・特殊文字の誤り）
- OCR: `送付電` → 参照: `送付管`（形が似た漢字の誤り）
- 全体的に高精度で実用レベル

### digidepo_3048008（68.2%）
- 昭和初期〜戦後直後の資料（表組み・縦書き混在）
- OCR: `昭和26年度 (第5回) 昭和27年度 (第5回)` → 参照: `昭和26年度(第4回)研修に引続き、昭和27年度(第5回)`（行の合体・テキストの欠落）
- 複雑なレイアウト（表・縦横混在）でレイアウト認識に課題
- 低精度の主因は行の合体・分割ミスによる読み順整序の乱れ

### digidepo_11048278（99.1%）
- 機関誌（2017年発行）の現代的印刷物
- ほぼ完璧な認識精度
- 現代の標準的な印刷物には非常に高い精度

## 総合評価

### 長所
- **GPU不要**でCPUのみで動作し、一般PCで実用的な速度（1〜2秒/ページ）
- **現代印刷物に対する精度が非常に高い**（99%超）
- **3形式同時出力**（TXT/XML/JSON）で後処理しやすい
- **ビジュアライズ機能**付きで認識状況を確認しやすい
- CC BY 4.0で商用利用も可能

### 短所・課題
- **複雑レイアウト（表・縦横混在）では精度低下**（68%程度）
- 旧字体・旧仮名（ヱ・ヲ等）の認識に一部誤りあり
- 起動時のモデルロードに4〜5秒かかる（バッチ処理では気にならない）

### 用途別推薦度
| 用途 | 推薦度 |
|---|---|
| 現代の図書・雑誌のデジタル化 | ★★★★★ |
| 昭和後期以降の公文書・報告書 | ★★★★☆ |
| 戦前・戦後直後の複雑レイアウト資料 | ★★★☆☆ |
| くずし字・古典籍 | ★★☆☆☆（専用の NDL古典籍OCR-Lite を推奨）|

---

## 結論

NDLOCR-Lite は「GPU不要の実用的な日本語OCR」という目標を十分に達成している。
現代の標準的な印刷物であれば99%超の精度でテキスト化でき、
一般的なノートPCでもページあたり1〜2秒という実用的な速度で動作する。

古い資料や複雑なレイアウトでは精度が落ちるが、それはOCR一般の課題であり、
このツールが特別に劣っているわけではない。図書館・アーカイブ系の
デジタル化プロジェクトや個人の資料整理に広く活用できる有用なツールである。
